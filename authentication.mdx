## API Key Authentication

The Cleanview API uses API key authentication. All requests must include a valid API key in the Authorization header.

### API Key Format

API keys follow this format:
```
cv_pub_[32 random characters base64url encoded]
```

**Example:** `cv_pub_KJ3x9mN2pQ8rS7tV5wY6zA1bC4dE0fG2`

### Making Authenticated Requests

Include your API key in the `Authorization` header using the Bearer token format:

```bash
Authorization: Bearer cv_pub_your_api_key_here
```

## Request Examples

<CodeGroup>

```bash cURL
curl -H "Authorization: Bearer cv_pub_your_api_key_here" \
  https://api.cleanview.energy/api/v0/eia-860m
```

```javascript JavaScript
const response = await fetch('https://api.cleanview.energy/api/v0/eia-860m', {
  headers: {
    'Authorization': 'Bearer cv_pub_your_api_key_here',
    'Content-Type': 'application/json'
  }
});

const data = await response.json();
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer cv_pub_your_api_key_here',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://api.cleanview.energy/api/v0/eia-860m',
    headers=headers
)

data = response.json()
```

```php PHP
<?php
$headers = [
    'Authorization: Bearer cv_pub_your_api_key_here',
    'Content-Type: application/json'
];

$ch = curl_init();
curl_setopt($ch, CURLOPT_URL, 'https://api.cleanview.energy/api/v0/eia-860m');
curl_setopt($ch, CURLOPT_HTTPHEADER, $headers);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

$response = curl_exec($ch);
curl_close($ch);

$data = json_decode($response, true);
?>
```

</CodeGroup>

## Authentication Errors

When authentication fails, the API will return a `401 Unauthorized` status code with an error response:

```json
{
  "success": false,
  "error": "INVALID_API_KEY",
  "message": "The provided API key is invalid or has been revoked"
}
```

### Common Error Codes

| Error Code | Description |
|------------|-------------|
| `API_KEY_MISSING` | No API key provided in the request |
| `INVALID_API_KEY` | API key doesn't exist or is inactive |
| `API_KEY_REVOKED` | API key has been revoked |

## Security Best Practices

<AccordionGroup>
  <Accordion icon="shield-check" title="Keep Your API Key Secure">
    - Never expose your API key in client-side code or public repositories
    - Store API keys as environment variables
    - Use different API keys for different environments (development, staging, production)
    - Regularly rotate your API keys
  </Accordion>
  
  <Accordion icon="server" title="Server-Side Only">
    API keys should only be used in server-side applications. Never include them in:
    - Frontend JavaScript code
    - Mobile applications
    - Public GitHub repositories
    - Configuration files committed to version control
  </Accordion>
  
  <Accordion icon="eye" title="Monitor Usage">
    - Regularly review your API usage patterns
    - Watch for unexpected spikes in requests
    - Contact support if you notice suspicious activity
  </Accordion>
</AccordionGroup>

## Getting Your API Key

<Steps>
  <Step title="Contact Our Team">
    During the beta phase, API keys are manually provisioned. Email [api@cleanview.energy](mailto:api@cleanview.energy?subject=API%20Key%20Request) to request access.
  </Step>
  
  <Step title="Provide Information">
    Include in your request:
    - Your name and company
    - Intended use case for the API
    - Expected request volume
    - Technical contact information
  </Step>
  
  <Step title="Receive Your Key">
    Our team will review your request and provide you with an API key along with any specific usage guidelines.
  </Step>
</Steps>
